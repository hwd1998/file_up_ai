## 一、全局设计规范

### 1.1 色彩规范

| 用途 | Tailwind Token | 色值 | 说明 |
| :--- | :--- | :--- | :--- |
| 主色 | `blue-600` | `#2563EB` | 按钮、链接、激活态 |
| 主色悬浮 | `blue-700` | `#1D4ED8` | 按钮 hover 态 |
| 成功 / 绿 | `green-500` | `#22C55E` | 监控看板"已上传"状态 |
| 警告 / 黄 | `amber-400` | `#FBBF24` | 监控看板"上传失败"状态 |
| 危险 / 红 | `red-500` | `#EF4444` | 监控看板"未上传"状态、错误提示 |
| 背景 | `gray-100` | `#F3F4F6` | 页面底色 |
| 白色容器 | `white` | `#FFFFFF` | 卡片 / 表格背景 |
| 边框 | `gray-200` | `#E5E7EB` | 卡片、表格、输入框边框 |
| 辅助文字 | `gray-400` | `#9CA3AF` | 占位符、提示说明 |

### 1.2 字体规范

| 级别 | Tailwind 类 | 用途 |
| :--- | :--- | :--- |
| 页面标题 | `text-xl font-semibold` | 各页头部标题 |
| 卡片标题 | `text-base font-semibold` | 功能区块标题 |
| 正文 | `text-sm font-normal` | 表格、表单文字 |
| 辅助说明 | `text-xs text-gray-400` | 占位符、提示文字 |

### 1.3 间距规范（Tailwind Spacing）

| 场景 | Tailwind 类 | 对应尺寸 |
| :--- | :--- | :--- |
| 页面内容区左右内边距 | `px-6` | 24px |
| 卡片内边距 | `p-4` | 16px |
| 表单字段垂直间距 | `space-y-4` | 16px |
| 按钮水平间距 | `gap-2` | 8px |
| 左侧边栏宽度 | `w-56` | 224px |

### 1.4 响应式

仅支持 PC Web（≥1280px 宽度），不做移动端适配。

**交互说明**：

- 管理员标识由后端 Session 注入，自动路由到对应首页

## 四、关键交互状态设计

### 4.1 上传任务状态流转（前端展示）

| 状态 | 图标 | 颜色 | 描述 |
| :--- | :--- | :--- | :--- |
| 等待中 | ⏳ | 灰色 | 文件排队等待上传 |
| 上传中 | 📤 | 蓝色 | 分片上传进行中，显示进度条 |
| 快速校验中 | 🔍 | 蓝色 | 同步校验表头/必填（2秒内） |
| 已暂停 | ⏸️ | 橙色 | 网络中断，显示"点击重试" |
| 快速校验失败 | ❌ | 红色 | 前端直接展示错误行列信息 |
| 处理中 | ⚙️ | 蓝色 | 异步完整校验+写入数据库中 |
| 成功 | ✅ | 绿色 | 数据写入成功 |
| 部分成功 | ⚠️ | 橙色 | 脏数据行被跳过，其余成功 |
| 校验失败 | ❌ | 红色 | 完整校验失败 |
| 失败 | ❌ | 红色 | 数据库写入失败（可重试） |

### 4.3 字段映射交互（Step2）

- 左列（文件字段）和右列（目标表字段）各为一排卡片
- 连线采用 SVG 绘制，鼠标悬浮连线高亮
- 拖拽左列字段名到右列字段名建立连线
- 已连线字段显示绿色，未连线显示灰色
- 右键菜单：删除连线 / 设置常量值 / 跳过此字段

### 4.4 全局通知提示（Toast）

自定义 Toast 组件，使用 Tailwind + Alpine.js 实现，挂载在 `<body>` 底部：

```html
<!-- 成功 Toast -->
<div class="fixed top-4 right-4 z-50 flex items-center gap-2
            bg-white border border-green-200 rounded-lg shadow-lg px-4 py-3"
     x-show="show" x-transition>
  <span class="text-green-500">✓</span>
  <span class="text-sm text-gray-700">操作成功</span>
</div>
```

| 类型 | 边框 / 图标颜色 | 自动消失 |
| :--- | :--- | :--- |
| 成功 | `border-green-200` / `text-green-500` | 2 秒 |
| 失败 | `border-red-200` / `text-red-500` | 5 秒，需手动关闭 |
| 异步结果通知 | `border-blue-200` / `text-blue-500` | 8 秒，右上角堆叠展示 |

## 六、前端技术实现说明

### 6.1 技术选型

| 技术 | 版本 | 引入方式 | 说明 |
| :--- | :--- | :--- | :--- |
| Thymeleaf | 3.x | Spring Boot 内置 | 模板渲染 |
| Tailwind CSS | 3.x | CDN Play / PostCSS 构建 | 全量样式工具类 |
| Alpine.js | 3.x | CDN | 轻量响应式交互（下拉、侧滑面板、Tab、折叠树） |
| Axios | 1.x | CDN | HTTP 请求 |
| SheetJS (xlsx) | 0.18+ | CDN | 前端读取 Excel 表头做同步快速校验 |
| SortableJS | 1.x | CDN | 拖拽排序（表单字段顺序调整） |
| SVG（原生） | — | 内联 | 字段映射连线绘制，无需额外库 |

> **说明**：Tailwind CSS 替代 Ant Design 后，不依赖 React/Vue，全部交互由 Alpine.js + 原生 JS 实现，与 Thymeleaf 模板配合零摩擦。

### 6.2 核心 Tailwind 组件封装清单

以下组件无法直接使用现成库，需在项目内自定义 Thymeleaf Fragment：

| 组件名 | Tailwind 实现要点 | Alpine.js 支持 |
| :--- | :--- | :--- |
| 侧滑面板（Slide-over） | `fixed inset-y-0 right-0` + `translate-x-full` 过渡 | `x-show` + `x-transition` |
| 模态框（Modal） | `fixed inset-0 bg-black/50` 遮罩 + 居中白色 `rounded-xl` | `x-show` |
| Toast 通知 | `fixed top-4 right-4 z-50` 堆叠列表 | `x-data` 管理队列 |
| 步骤条（Steps） | flex + 连接线 `border-t-2` + 激活态 `bg-blue-600` | 纯 Tailwind 静态 |
| 目录树 | 递归 Thymeleaf `th:fragment` + `pl-4` 缩进 | `x-show` 控制折叠 |
| 文件拖拽区 | `border-2 border-dashed border-gray-300` + `dragover:border-blue-500` | 原生 JS drag events |
| 进度条 | `bg-gray-200 rounded-full` + 内层 `bg-blue-600` 动态 `width` | Alpine.js `x-bind:style` |
| 开关（Toggle） | `relative inline-flex` + `translate-x` 滑块 | `x-model` |
| 下拉选择（Select） | 原生 `<select>` + `appearance-none` Tailwind 美化 | 原生 |
| 级联选择器 | 多列 `<select>` 联动 | Alpine.js 联动数据 |

### 6.3 页面复杂度分级

| 复杂度 | 页面 | 说明 |
| :--- | :--- | :--- |
| 高 | 模板创建向导 Step2（字段映射） | SVG 内联连线 + Alpine.js 拖拽状态管理 |
| 高 | 文件上传页（大文件分片） | Web Worker 分片 + 断点续传 |
| 中 | 模板创建向导 Step3/4 | 拖拽排序 + 动态表单 + Alpine.js 开关联动 |
| 中 | 监控看板 | 递归树 + 状态色块 |
| 低 | 历史列表、目录管理、告警配置 | 标准 CRUD，Tailwind 表格 + 表单 |

### 6.3 分片上传前端逻辑

```
1. 选择文件 → 同步表头校验（读取前10行 SheetJS）
2. 校验通过 → 切片（每片 5MB）→ 并发上传（最多3片同时）
3. 每片上传成功 → 更新进度条
4. 全部片段上传完成 → 调用合并接口
5. 网络中断 → 记录已上传片段 Index → 续传从断点开始
```
