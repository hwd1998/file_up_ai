## 一、项目背景与目标

### 背景

国货美妆集团各业务部门（产品营销、渠道营销、品牌等）需定期向数据中台上报经营/营销数据，当前存在：

- 填报不及时
- 文件格式错误、表头不规范
- 学习成本高，数据异常频发

### 目标

建设统一的**文件上传系统**，实现：

- **运营自助**：业务用户无需 IT 介入，按模板上传文件即可完成数据上报
- **模板化管理**：IT 管理员灵活配置模板（字段映射、校验规则、版本管理），支持一键建表
- **可观测性**：按目录/业务线隔离，提供上传监控、飞书告警、历史追溯
- **中台对接**：数据写入 MySQL，由数据中台定时抽取，不触发下游调度

### 核心 KPI

| 指标 | 目标值 |
|------|--------|
| 业务用户自助上传成功率 | ≥ 95% |
| 模板配置后 IT 介入次数 | 0 次（除模板变更） |
| 上传监控告警覆盖率 | 100% |
| 2GB 文件上传完成时间 | ≤ 10 分钟（标准办公网） |
| 文件保留时长 | 3 个月后自动清理 |

## 二、用户角色与权限

### 角色定义

| 角色 | 说明 | 典型操作 |
|------|------|----------|
| IT 管理员 | 负责系统配置、模板管理、监控规则 | 创建/编辑模板、配置校验规则、管理目录/用户授权、查看监控看板、设置告警 |
| 业务用户 | 各业务线/品牌/部门上报人员 | 在授权目录下上传文件、查看自己的上传历史、接收飞书通知 |

业务用户可按组织层级细分（如"产品营销-精华类目"、"渠道营销-抖音渠道"），通过**目录结构**实现权限隔离。

### 账号体系

- 飞书 OAuth2.0 登录，系统不存储密码
- 首次登录自动创建账号并同步飞书组织架构
- IT 管理员可根据飞书部门自动或手动分配目录权限
- 用户离职后飞书账号失效即无法登录

### 目录与授权

- 支持多级目录结构（如"集团/美妆事业部/产品营销/2026年活动数据"）
- 每个**最下级目录**绑定唯一一个已发布模板
- 授权粒度：按目录授权"上传"和"查看历史"权限，支持批量授权给用户或飞书部门

## 三、功能需求

### 3.1 模板管理（IT 管理员）

#### 模板创建向导（5 步）

| 步骤 | 功能点 | 详细要求 |
|------|--------|----------|
| 1. 基础信息 | 模板名称、所属目录、描述 | 模板名称在目录内唯一，一个目录只能绑定一个模板 |
| 2. 样例上传与建表 | 上传样例文件（Excel/CSV，≤100 行） | 自动解析字段类型，支持**一键生成目标 MySQL 表**（先展示 DDL 可编辑后执行） |
| 3. 上传表单配置 | 配置上传时除文件外的必填元数据字段 | 支持文本/数字/日期/单选/多选/下拉，可配置是否必填、默认值、选项列表 |
| 4. 校验规则配置 | 表级 + 字段级规则 | 详见 3.2 节，支持从已有模板复制规则集 |
| 5. 完成与发布 | 保存模板，生成 V1.0 版本 | 发布后不可删除，仅可停用或新建版本 |

#### 模板版本管理

- 版本号系统自动递增（V1.0、V1.1、V2.0...）
- 业务用户始终使用**最新发布版本**
- 版本变更记录变更日志（操作人、时间、变更内容），**不可删除**

#### 上传周期配置

- 可配置上传周期：日 / 周 / 月
- 配置当前传数执行人，用于提醒和监控
- 支持"本周期无数据"选项，无需上传文件

#### 模板增删改

- 模板名称唯一，具备增删改停用功能
- 已启用模板须先停用再删除

### 3.2 校验规则配置

| 校验类别 | 说明 | 是否必要 |
|----------|------|----------|
| 表头字段名称匹配 | 列名须与模板一致（大小写敏感可配） | **必要** |
| 必填列 | 指定字段不得为空 | **必要** |
| 字段类型校验 | 数值/日期/文本等类型匹配 | **必要** |
| 枚举值校验 | 字段值限定在预定义列表内 | **必要** |
| 正则表达式校验 | 支持常用模板及自定义正则 | **必要** |
| 数值范围校验 | 最小值/最大值限制 | 可选 |
| 跨列依赖校验 | 如开始日期 ≤ 结束日期 | 可选 |
| 唯一性约束 | 文件内不允许重复行（单列或复合列） | 可选 |
| 唯一键去重策略 | 与目标表冲突时：跳过 / 覆盖更新 / 报错（整批拒绝） | 可选 |
| 脏数据阈值 | 超阈值整批拒绝，未超则跳过错误行继续导入 | 可选 |

**校验执行顺序**：表结构 → 字段类型/格式/枚举 → 跨列依赖 → 唯一性（文件内）→ 唯一键（与目标表冲突）

### 3.3 文件上传（业务用户）

#### 上传流程

1. 飞书扫码/授权登录系统
2. 进入授权目录树，点击最下级目录
3. 拖拽或点击上传文件
4. 系统立即执行**同步校验**（表头、必填列、简单类型）；失败则提示具体错误（第 XX 行/列有什么问题），不上传
5. 同步校验通过后，文件上传至临时存储，生成**任务 ID**，状态为"校验中"
6. 后端**异步执行完整校验**，通过后写入数据库，更新任务状态
7. 用户可关闭页面，完成后通过**飞书消息**通知结果

#### 文件要求

| 属性 | 要求 |
|------|------|
| 支持格式 | .xls、.xlsx、.csv |
| 单文件大小上限 | 2 GB |
| Excel 单 Sheet 行数 | ≤ 100 万行（超出拒绝并提示） |
| 批量上传 | 单次最多 40 个文件，每个文件独立校验 |
| 上传方式 | Web 界面拖拽或点击，HTTPS 加密传输，显示单个文件上传进度 |

#### 断点续传

- 网络中断自动暂停，重试后从断点继续（分片上传）
- 服务端异步处理失败自动重试最多 3 次；业务校验失败不重试

### 3.4 字段映射与一键建表

- 样例文件解析字段后，系统推断数据类型，先弹窗展示 DDL 语句，**允许编辑后再执行**
- 映射界面支持：自动同名连线、手动拖拽连线、常量映射、跳过字段
- 支持自定义表名、索引、字符集等高级选项

### 3.5 异步处理与存储

**处理流程**：

```
用户上传 → 同步校验（表头/必填）→ 生成任务 → 异步完整校验
→ 写入 MySQL（数据）+ 写入 OSS（原始文件副本）→ 更新任务状态 → 飞书通知
```

**存储策略**：

| 存储介质 | 内容 | 保留期 |
|----------|------|--------|
| MySQL | 任务记录、元数据、校验错误明细 | 3 个月（可配置延长） |
| MySQL | 实际导入的业务数据表 | 永久（由数据中台管理） |
| OSS | 原始上传文件副本（供业务核对下载） | 3 个月 |

> 数据中台定时从 MySQL 及 OSS 拉取数据，本系统不主动推送。

### 3.6 上传历史与监控

#### 业务用户视角（上传历史列表）

字段：文件名、所属目录、模板名及版本、上传人、上传时间、文件大小、数据行数、处理状态、失败原因、操作（查看详情/下载源文件/删除（仅失败记录））

处理状态枚举：`等待校验` / `校验中` / `校验失败` / `处理中` / `成功` / `部分失败`

支持按时间范围、状态、文件名模糊搜索，分页每页 20 条。

#### IT 管理员视角（监控看板）

| 上报状态 | 定义 | 颜色 |
|----------|------|------|
| 未上传 | 周期内无成功上传记录 | 红色 |
| 上传失败 | 有记录但最终状态为校验失败/部分失败 | 黄色 |
| 上传成功 | 至少一条成功记录且无失败记录 | 绿色 |

- 按目录树展示，支持展开/折叠
- 点击红/黄单元格可查看具体用户和失败原因
- 支持导出监控报表（Excel）

**定时提醒**：截止时间前飞书推送给业务用户及其上级，内容：`【数据上报提醒】您负责的【目录名称】在【周期】尚未上传，请及时处理`

### 3.7 告警与通知

#### 飞书通知场景

| 场景 | 接收人 | 消息内容要素 |
|------|--------|-------------|
| 异步处理失败 | 上传者本人 | 任务 ID、文件名、失败原因（行级错误摘要）、失败时间 |
| 未及时上传（监控触发） | 业务用户 + 目录管理员 | 目录名称、周期、截止时间 |
| 上传失败率突增（系统告警） | IT 管理员飞书群 | 时间窗口、失败率、失败任务数 |
| 模板校验规则执行异常 | IT 管理员 | 模板名、异常堆栈 |

#### 告警阈值

| 监控项 | 阈值 | 告警等级 |
|--------|------|----------|
| 最近 1 小时上传失败率 | > 10% | 严重 |
| 单文件解析耗时 | > 5 分钟 | 警告 |
| 异步处理队列堆积数 | > 200 个任务 | 警告 |
| 并发上传用户数 | > 60 人 | 提示 |
| MySQL/OSS 写入连续失败次数 | 连续 3 次 | 严重 |

## 四、非功能需求

### 性能与容量

| 指标 | 要求 |
|------|------|
| 并发上传用户数 | ≥ 50 |
| 单文件最大上传时间（2GB） | ≤ 10 分钟（办公网） |
| 异步处理吞吐 | 单节点 ≥ 10 个文件/秒（含校验+写入） |
| 历史列表查询响应 | ≤ 2 秒（百万级记录） |
| 数据预览响应（前 10 行） | ≤ 1 秒 |
| 同步校验（表头/必填） | ≤ 2 秒（文件上传后） |

### 可用性与容错

- 服务无状态设计，支持多节点水平扩展
- 异步处理队列持久化（RabbitMQ + 数据库任务表），服务重启不丢任务
- 数据库连接池重试机制

### 安全

- 全站 HTTPS（TLS 1.2+）
- 认证完全委托飞书，本系统不存储密码
- 目录级数据隔离，用户只能访问被授权的目录

### 可观测性

- 提供关键指标（上传 QPS、失败率、队列长度），可视化监控面板（Grafana 或自研）
- 集成飞书告警（Webhook）
- 所有操作日志（模板变更、权限变更、用户登录）记录到审计表，保留 1 年

## 五、验收标准（核心场景）

| 场景 | 预期结果 |
|------|----------|
| IT 管理员创建新模板：上传样例→配置校验→发布 | 15 分钟内完成，状态"已发布"，MySQL 建表成功，目录与模板绑定 |
| 业务用户飞书登录→上传合规 Excel（10 万行） | 同步校验通过，异步处理成功，飞书收到成功通知，历史状态"成功" |
| 上传缺少必填列的文件 | 同步校验失败，前端提示"缺少列：XX"，文件未被接收 |
| 上传含唯一键重复的文件（文件内重复） | 校验失败，整批拒绝，错误明细显示重复行号 |
| 批量上传 40 个小文件 | 全部异步处理成功，历史列表 40 条记录状态均"成功" |
| 上传过程中网络中断 | 前端暂停，重试后从断点继续，不产生脏数据 |
| 跨目录数据隔离验证 | 用户 B 看不到用户 A（不同目录）的上传记录 |
| 异步处理失败（数据库写入超时） | 飞书通知用户，自动重试 3 次后仍失败则最终失败 |
| 监控周期结束某目录未上传 | 看板显示红色，飞书提醒发送给用户及管理员 |
| 超过 3 个月的文件 | 自动删除，下载按钮置灰，元数据保留 |
| 上传失败率突增至 15% | 触发严重告警，飞书群通知 IT 管理员 |

## 六、约束与假设

- 上传文件不含强敏感数据（身份证、银行卡），暂不做脱敏
- 数据中台负责从 MySQL/OSS 拉取数据，本系统不主动推送
- 所有业务用户均有飞书企业账号
