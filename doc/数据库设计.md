# 文件上传系统 — 数据库设计

## 总览

| 分层 | 表 | 说明 |
|------|-----|------|
| 目录与权限 | `directories` | 多级目录树 |
|  | `directory_permissions` | 用户目录权限 |
| 模板配置 | `templates` | 模板主表 |
|  | `template_versions` | 模板版本 |
|  | `template_fields` | 模板字段定义（文件列映射） |
| 上传任务 | `upload_tasks` | 上传任务记录 |
|  | `upload_errors` | 校验错误明细 |
| 业务数据 | `biz_*` | 每个模板对应一张，建模板时创建 |

---

## 一、目录与权限

### `directories` 目录树

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| parent_id | INT | 父目录 ID，顶级为 NULL |
| name | VARCHAR | 目录名称 |
| full_path | VARCHAR | 完整路径，如 /集团/美妆/产品营销 |
| template_id | INT | 绑定的模板 ID，仅最下级目录有值 |
| created_by | INT | 创建人 |
| created_at | DATETIME | 创建时间 |

### `directory_permissions` 目录权限

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| directory_id | INT | 目录 ID |
| user_id | INT | 用户 ID |
| permission_type | VARCHAR | upload / view_history |

---

## 二、模板配置

### `templates` 模板主表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| name | VARCHAR | 模板名称（目录内唯一） |
| directory_id | INT | 所属最下级目录 |
| status | VARCHAR | active / disabled |
| current_version | VARCHAR | 当前最新版本号，如 V1.2 |
| form_fields | JSON | 上传表单配置（见示例） |
| created_by | INT | 创建人 |
| created_at | DATETIME | 创建时间 |

`form_fields` JSON 示例：
```json
[
  {"name": "费用发生月份", "type": "date", "required": true},
  {"name": "投放工具", "type": "select", "required": true, "options": ["抖音", "快手", "视频号"]},
  {"name": "备注", "type": "text", "required": false}
]
```

### `template_versions` 模板版本

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| template_id | INT | 所属模板 ID |
| version | VARCHAR | 版本号，如 V1.0 / V1.1 |
| target_table_name | VARCHAR | 对应的业务数据表名，如 biz_产品营销_001 |
| change_log | TEXT | 版本变更说明 |
| created_by | INT | 操作人 |
| created_at | DATETIME | 创建时间 |

### `template_fields` 模板字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| version_id | INT | 所属版本 ID |
| field_name | VARCHAR | 字段显示名称 |
| source_column | VARCHAR | 文件中的列名 |
| target_column | VARCHAR | 目标表中的列名 |
| field_type | VARCHAR | 字段类型，如 int / varchar / date |
| field_order | INT | 列顺序 |
| is_required | BOOLEAN | 是否必填 |
| validation_rules | JSON | 校验规则（见示例） |

`validation_rules` JSON 示例：
```json
{
  "enum": ["抖音", "快手", "视频号"],
  "regex": null,
  "min": null,
  "max": null
}
```

## 三、上传任务

### `upload_tasks` 上传任务记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| template_id | INT | 模板 ID |
| version_id | INT | 使用的模板版本 ID |
| file_name | VARCHAR | 原始文件名 |
| file_size | BIGINT | 文件大小（字节） |
| file_path | VARCHAR | 服务器存储路径（保留3个月） |
| status | VARCHAR | pending / validating / success / failed / partial_failed |
| row_count | INT | 成功写入行数 |
| uploaded_by | INT | 上传人用户 ID |
| form_data | JSON | 上传时填写的表单数据（见示例） |
| created_at | DATETIME | 上传时间 |
| finished_at | DATETIME | 处理完成时间 |

`form_data` JSON 示例：
```json
{
  "费用发生月份": "2026-03",
  "投放工具": "抖音",
  "备注": ""
}
```

### `upload_errors` 校验错误明细

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| task_id | INT | 所属任务 ID |
| row_number | INT | 出错行号 |
| column_name | VARCHAR | 出错列名 |
| error_type | VARCHAR | 错误类型，如 type_mismatch / enum_invalid |
| error_message | VARCHAR | 错误描述，如"第3行渠道字段值不在枚举范围内" |

## 四、业务数据表（动态建表）

每个模板在发布时，由管理员通过"一键建表"在数据库中创建对应的业务表。

- 表名存在 `template_versions.target_table_name` 中
- 表结构与模板字段一一对应
- 数据中台直接从这些表抽取数据
- 本系统不负责删除，由数据中台管理

示例：
```sql
CREATE TABLE biz_产品营销数据_001 (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  task_id     INT,          -- 关联 upload_tasks.id，可追溯来源
  _version    VARCHAR(10),  -- 写入时的模板版本号，如 V1.0 / V2.0
  渠道        VARCHAR(50),
  费用        DECIMAL(12,2),
  月份        DATE,
  created_at  DATETIME
);
```

## 数据保留策略

| 数据 | 保留时长 | 清理方式 |
|------|----------|----------|
| 原始文件（file_path） | 3个月 | 定时任务删除文件，按钮置灰 |
| upload_tasks / upload_errors | 3个月 | 定时任务删除记录 |
| 业务数据表（biz_*） | 永久 | 数据中台管理，本系统不删 |
